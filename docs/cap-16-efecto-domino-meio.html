16. El Efecto Dominó

En el capítulo anterior, construimos un laboratorio para un solo producto en una sola ubicación. Pero en la realidad, el inventario no vive en islas aisladas. Una decisión de compra en tu Centro de Distribución (CD) afecta la disponibilidad de tus tiendas, y un retraso en la fábrica genera ondas de choque que terminan en estantes vacíos a miles de kilómetros de distancia.

Imagina que mueves ligeramente una cuerda en un extremo: el movimiento es casi imperceptible al inicio, pero se amplifica violentamente al llegar al otro extremo. Bienvenidos a la gestión de Redes Complejas. En este capítulo, exploraremos cómo la variabilidad se propaga por la red y cómo el Arquitecto de Suministros utiliza la Optimización Multi-Echelon (MEIO) para poner el stock donde realmente aporta valor.

16.1 El Efecto Látigo (Bullwhip Effect)

El Efecto Látigo es, probablemente, el fenómeno más destructivo y costoso en la Supply Chain moderna. Describe cómo pequeñas y naturales fluctuaciones en la demanda del cliente final se transforman en oscilaciones masivas, caóticas e impredecibles para el fabricante.

¿Por qué ocurre? La anatomía de la distorsión

Este fenómeno no es un error de sistema, sino una consecuencia sistémica de cuatro comportamientos profundamente humanos y operativos:

  1. Actualización de Pronósticos en Cascada: Cada eslabón de la cadena (Tienda -> Mayorista -> CD -> Fábrica) realiza su propio pronóstico basado en los pedidos que recibe de su cliente inmediato, no en la venta real al consumidor final. Si la tienda sube su pedido un 5%, el CD entra en pánico y lo sube un 10%, y la fábrica termina duplicando su capacidad. La señal original se pierde en el “ruido” de los intermediarios.

  2. Loteo de Pedidos (Batching): Por razones de eficiencia económica (el EOQ que vimos en el Cap. 12), las empresas no piden unidades individuales cada vez que venden una. Esperan a llenar un camión o un pallet. Esto genera “pulsos” de demanda: periodos de calma total seguidos de explosiones de pedidos que saturan la capacidad de producción.

  3. Fluctuación de Precios y Promociones: Las ofertas de “comprar por adelantado” o descuentos por volumen incitan a los clientes a realizar compras especulativas. Esto crea picos de demanda artificiales que no reflejan el consumo real, engañando a los modelos estadísticos de la fábrica.

  4. El Juego de la Escasez (Shortage Gaming): Cuando un producto escasea, los compradores tienden a inflar sus pedidos con la esperanza de que, tras el prorrateo, les llegue lo que realmente necesitan. Esto crea una demanda ficticia que desaparece súbitamente cuando el stock se normaliza, dejando a la fábrica con un exceso masivo de inventario.

Consecuencias: El Efecto Látigo drena la rentabilidad al exigir mayor capacidad de almacenamiento, fletes urgentes costosos y una utilización de planta ineficiente.

16.2 Optimización Local vs. Global (MEIO)

La mayoría de las empresas operan bajo un modelo de Optimización Descentralizada. En este esquema, el gerente de la tienda y el gerente del CD tienen sus propios KPI de nivel de servicio y stock de seguridad. El problema es que esta “seguridad” se duplica innecesariamente.

  • El Problema: Si la tienda mantiene stock para cubrirse del CD, y el CD mantiene stock para cubrirse de la Fábrica, la red termina con una acumulación de inventario redundante. Es una “suma de miedos” que el balance general debe pagar.

La Optimización Multi-Echelon (MEIO) rompe esta inercia al mirar la red como un solo organismo. Su objetivo no es optimizar cada bodega, sino responder: ¿Cuál es la combinación mínima de inventario en TODA la red que garantiza el nivel de servicio al cliente final? A veces, la respuesta es quitar stock de las tiendas y centralizarlo en un “hub” regional, o viceversa.

16.3 El Poder del “Risk Pooling”

El Risk Pooling es la magia estadística de la agregación y es la base de la centralización logística. La demanda de un cliente individual es salvaje e impredecible; pero la demanda de un grupo de 100 clientes es estable.

La Regla de la Raíz Cuadrada: Matemáticamente, si tienes \(n\) bodegas independientes y decides centralizar todo su inventario en una sola ubicación centralizada, el Stock de Seguridad (\(SS\)) necesario se reduce teóricamente por un factor de \(\sqrt{n}\).

  • Ejemplo Numérico: Si tienes 9 tiendas regionales y cada una requiere 100 unidades de seguridad para un 95% de nivel de servicio (Total: 900 unidades), al centralizar la red en un solo CD nacional solo necesitarías \(\frac{900}{\sqrt{9}} = 300\) unidades para garantizar el mismo nivel de protección. ¡Una reducción del 66% del capital inmovilizado!

16.4 Simulador: Propagación del Efecto Látigo

En esta simulación, puedes observar en tiempo real cómo la variabilidad que siente la Fábrica es exponencialmente mayor que la que siente la Tienda. Ajusta la incertidumbre del cliente y mira cómo el “latigazo” se vuelve incontrolable.

jStat = require("jstat@1.9.6")
Plotly = require("plotly.js-dist-min@2.24.1")

viewof retail_sigma = Inputs.range([1, 20], {value: 5, step: 1, label: "Incertidumbre Cliente (σ):"})
viewof batch_size = Inputs.range([1, 100], {value: 20, step: 5, label: "Tamaño de Lote (Tienda):"})
viewof lead_time_net = Inputs.range([1, 10], {value: 3, step: 1, label: "Lead Time entre eslabones:"})
network_sim = {
  const days = 60;
  const mean_demand = 50;
  
  let store_stock = 200;
  let cd_stock = 500;
  
  const history = [];
  let pending_to_store = [];
  let pending_to_cd = [];

  for (let t = 0; t < days; t++) {
    // 1. Demanda Real del Cliente
    const actual_demand = Math.max(0, jStat.normal.sample(mean_demand, retail_sigma));
    
    // 2. Operación de Tienda
    store_stock -= actual_demand;
    let store_order = 0;
    if (store_stock < 100) { 
      store_order = Math.ceil((150 - store_stock) / batch_size) * batch_size;
      pending_to_store.push({arrival: t + lead_time_net, qty: store_order});
    }
    
    // 3. El CD recibe el pedido de la Tienda (Señal amplificada)
    let cd_order = 0;
    if (store_order > 0) {
      cd_stock -= store_order;
      if (cd_stock < 300) {
        cd_order = Math.ceil((500 - cd_stock) / (batch_size * 2)) * (batch_size * 2);
        pending_to_cd.push({arrival: t + lead_time_net, qty: cd_order});
      }
    }

    // Procesar llegadas
    pending_to_store = pending_to_store.filter(o => {
      if (o.arrival <= t) { store_stock += o.qty; return false; }
      return true;
    });

    history.push({
      day: t,
      client: actual_demand,
      store_order: store_order,
      cd_order: cd_order
    });
  }
  return history;
}

WarningObservación Estratégica

Nota cómo la demanda del cliente (línea gris) es relativamente estable, pero las órdenes que llegan a la fábrica (barras rojas) son violentas y esporádicas. Esto es lo que destruye la eficiencia de producción y dispara los costos de transporte de emergencia.

16.5 Estrategias para Mitigar el Dominó

Como Arquitecto de Suministros, tienes tres herramientas fundamentales para “domar” el látigo y devolver la cordura a la cadena:

  1. Visibilidad de la Demanda (VMI / CPFR): Compartir el dato de venta real (POS) con toda la cadena. Si la fábrica sabe exactamente lo que vendió la tienda hoy, puede planificar su producción sin esperar a que el CD le envíe un pedido inflado. La información reemplaza al inventario.

  2. Reducción radical de Lead Times: Cuanto más corto es el tiempo de respuesta, menos stock de seguridad se necesita en cada eslabón. Si un proveedor tarda 1 día en lugar de 30, la incertidumbre desaparece casi por completo.

  3. Sincronización de Lotes y “Everyday Low Price” (EDLP): Eliminar las promociones agresivas que causan compras especulativas y trabajar en reducir el costo de pedido para que los lotes sean más pequeños y frecuentes, suavizando el flujo de mercancía.
