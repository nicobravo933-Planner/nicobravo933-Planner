19. El Algoritmo vs. el Ego: Gestión de Sesgos Humanos

En el capítulo anterior, aprendimos a utilizar el Forecast Value Added (FVA) para medir objetivamente si las intervenciones humanas mejoran o destruyen la precisión del sistema. Pero medir el daño no es lo mismo que prevenirlo. Para un Arquitecto de Suministros, entender la matemática de los modelos es solo la mitad del trabajo; la otra mitad es entender la psicología del planificador.

A menudo, los ajustes manuales que destruyen valor no nacen de una mala intención, sino de fallos estructurales en la forma en que el cerebro humano procesa la incertidumbre. En este capítulo, exploraremos los sesgos cognitivos que sabotean el pronóstico y cómo diseñar un proceso que proteja al algoritmo del ego.

19.1 El Cerebro no es Estadístico

Evolutivamente, nuestro cerebro está diseñado para detectar patrones rápidos para la supervivencia, no para calcular regresiones lineales o distribuciones Gamma en la cabeza. Esto nos hace propensos a “ver” tendencias donde solo hay ruido aleatorio.

Cuando un gerente de ventas dice: “Siento que la demanda va a explotar este mes”, su cerebro está siendo víctima de una serie de atajos mentales o heurísticas. Como arquitectos, nuestra misión no es eliminar el juicio humano (que es valioso para eventos únicos), sino “higienizarlo”.

19.2 Los Sesgos más Peligrosos en Supply Chain

Existen cuatro sesgos que actúan como “virus” en el proceso de planificación:

  1. Sesgo de Recencia (Recency Bias): Es la tendencia a darle un peso excesivo a los eventos más recientes. Si ayer un cliente importante canceló un pedido, el planificador tiende a bajar el pronóstico de todo el mes, ignorando que los últimos dos años de datos muestran una tendencia alcista sólida.

  2. Efecto de Anclaje (Anchoring): Ocurre cuando nos aferramos a un número inicial (como el presupuesto anual) y ajustamos el pronóstico basándonos en ese “ancla”, en lugar de mirar la realidad del mercado.

  3. Sesgo de Confirmación: Buscamos información que respalde nuestra corazonada. Si creemos que un producto va a fallar, solo prestaremos atención a las noticias negativas, ignorando las señales positivas del modelo de Machine Learning.

  4. Sesgo de Optimismo (Over-optimism): El más común en las áreas comerciales. Se confunde el deseo de vender con la probabilidad de vender. Esto genera el “ajuste al alza” sistemático que infla los inventarios de seguridad.

19.3 El Campo de Batalla: La reunión de S&OP

La reunión de Consenso o S&OP suele ser el lugar donde el ego derrota al algoritmo. Los factores sociales amplifican los sesgos individuales:

  • Groupthink (Pensamiento de Grupo): Nadie quiere contradecir al director general cuando dice que la nueva campaña será un éxito rotundo, incluso si los datos dicen lo contrario.

  • Jerarquía vs. Evidencia: En muchas culturas organizacionales, el “HiPPO” (Highest Paid Person’s Opinion) pesa más que el RMSE del modelo de Random Forest.

Estrategia del Arquitecto: Para combatir esto, el pronóstico estadístico debe presentarse como la “Hipótesis Nula”. Cualquier cambio humano debe estar justificado por una evidencia externa que el modelo no conozca (ej. un contrato firmado, una huelga portuaria). Si no hay evidencia, el modelo manda.

19.4 Simulador: El Test del Sesgo

Ponte a prueba. Imagina que el modelo estadístico predice una demanda estable. De repente, ocurren eventos externos. Ajusta el pronóstico y observa si tu “intuición” está añadiendo valor o simplemente reaccionando al ruido.

jStat = require("jstat@1.9.6")
Plotly = require("plotly.js-dist-min@2.24.1")

// Parámetros de la simulación
viewof event_type = Inputs.select(
  ["Venta récord ayer", "Queja de cliente clave", "Presión de cierre de mes", "Noticia de huelga"], 
  {label: "Evento del día:"}
)
viewof user_adjustment = Inputs.range([-40, 40], {value: 15, step: 5, label: "Tu ajuste manual (%):"})
actual_effect = {
  if (event_type === "Venta récord ayer") return 2; // Efecto real mínimo (ruido)
  if (event_type === "Queja de cliente clave") return -5;
  if (event_type === "Presión de cierre de mes") return 0; // Efecto nulo en demanda real
  if (event_type === "Noticia de huelga") return -20;
  return 0;
}

// Cálculo de Valor Agregado
baseline = 100;
human_guess = baseline * (1 + user_adjustment/100);
reality = baseline * (1 + actual_effect/100);

error_base = Math.abs(reality - baseline);
error_human = Math.abs(reality - human_guess);
fva = error_base - error_human;

19.5 Estrategias para Des-sesgar (De-biasing)

Para construir una organización resiliente, el Arquitecto debe implementar estas tres protecciones:

  1. Ajustes “Ciegos”: No permitas que el equipo comercial vea el número estadístico antes de dar su opinión. Esto evita el efecto de anclaje.

  2. Responsabilidad del Ajuste: Cada cambio manual debe llevar una “etiqueta” (ej. Promoción, Huelga, Intuición). Al final del año, el FVA dirá qué etiquetas son rentables.

  3. Rangos de Tolerancia: Si el ajuste humano es menor al 5%, el sistema lo ignora automáticamente. El ruido pequeño no merece el costo administrativo del cambio.

El éxito de una arquitectura de suministros no se mide por la complejidad de sus ecuaciones, sino por su capacidad de mantener la disciplina frente a la presión emocional del negocio.
