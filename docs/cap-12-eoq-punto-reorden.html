12. El Paso del Pronóstico a la Orden

En las secciones anteriores de este libro, nos obsesionamos con la precisión. Aprendimos a diseccionar la demanda utilizando desde suavizamiento exponencial hasta algoritmos de Machine Learning. Sin embargo, en la trinchera de la Supply Chain, el pronóstico es simplemente una “declaración de intenciones”. La realidad financiera solo se materializa cuando el camión llega al muelle de descarga y el inventario se asienta en el balance como capital inmovilizado.

Muchos planificadores operan bajo “reglas de dedo” heredadas: “Pedimos cada lunes” o “Mantenemos dos semanas de stock”. Estas heurísticas ignoran el equilibrio de costos que modelamos en el Capítulo 11. En este capítulo, estableceremos la física básica del inventario mediante la Fórmula de Wilson (EOQ) y el Punto de Reorden (ROP).

12. El Paso del Pronóstico a la Orden

12.1 El Escenario Determinista: Las Reglas del Juego

Para entender la base de la optimización, primero trabajaremos en un “laboratorio determinista”. Aquí, el mundo es predecible, lo que nos permite aislar las fuerzas que mueven el costo total:

  1. Demanda Constante (\(D\)): Sabemos exactamente cuántas unidades se venderán y la tasa es uniforme.

  2. Lead Time Invariante (\(L\)): El proveedor siempre tarda exactamente lo mismo.

  3. Costo de Pedido Fijo (\(S\)): Cada orden genera un gasto administrativo y logístico constante (flete, facturación, recepción).

  4. Costo de Mantenimiento (\(h\)): El costo calculado en el capítulo anterior.

12.2 El Tamaño de Lote Económico (EOQ)

El EOQ (Economic Order Quantity) responde a la pregunta: ¿Cuánto pedir?. Es el punto de equilibrio donde la eficiencia administrativa (pedir poco frecuente) se encuentra con la eficiencia financiera (no tener exceso de stock).

12.2.1 El Conflicto de Costos: La Gran Balanza

En la gestión de inventarios, enfrentamos un conflicto de intereses financieros:

  • Costo de Pedir (Curva Descendente): A medida que el tamaño del lote (\(Q\)) aumenta, pedimos menos veces al año, por lo que el costo total de pedidos cae.

  • Costo de Mantener (Curva Ascendente): A medida que \(Q\) aumenta, el inventario promedio en bodega crece, elevando los costos de capital y almacenaje.

El EOQ es precisamente el punto donde estas dos curvas se cruzan.

Plotly = require("plotly.js-dist-min@2.24.1")

viewof d_annual = Inputs.range([1000, 20000], {value: 5000, step: 100, label: "Demanda Anual (D):"})
viewof s_cost = Inputs.range([10, 500], {value: 100, step: 10, label: "Costo de Pedido (S):"})
viewof unit_v = Inputs.range([1, 200], {value: 50, step: 1, label: "Valor Unitario (v):"})
viewof h_rate = Inputs.range([0.05, 0.50], {value: 0.25, step: 0.01, label: "Tasa h (%):"})
viewof lead_time = Inputs.range([1, 30], {value: 10, step: 1, label: "Lead Time (días):"})
h_annual = unit_v * h_rate
eoq = Math.sqrt((2 * d_annual * s_cost) / h_annual)
d_daily = d_annual / 365
rop = d_daily * lead_time
cycle_days = (eoq / d_daily)

// Datos para la gráfica de cruce de curvas
cost_curves_data = {
  const points = [];
  const startQ = Math.max(10, eoq * 0.2);
  const endQ = eoq * 2.5;
  for (let q = startQ; q <= endQ; q += (endQ - startQ) / 50) {
    const c_order = (d_annual / q) * s_cost;
    const c_hold = (q / 2) * h_annual;
    points.push({ q, c_order, c_hold, c_total: c_order + c_hold });
  }
  return points;
}

La Fórmula Maestra

La intersección de estas dos curvas nos entrega la Raíz Cuadrada de Wilson:

$$EOQ = \sqrt{\frac{2 \times D \times S}{h}}$$

Donde:

  • \(D\): Demanda anual.

  • \(S\): Costo fijo de realizar un pedido.

  • \(h\): Costo de mantenimiento anual por unidad (\(v \times \text{tasa } h\)).

El Stock de Ciclo: En este modelo, tu inventario sigue un patrón de “dientes de sierra”. Tu Inventario Promedio es exactamente \(EOQ / 2\).

12.3 El Punto de Reorden (ROP): ¿Cuándo pedir?

El ROP es el nivel de inventario que activa la señal de compra. Debe cubrir la demanda durante el Lead Time.

$$ROP = d \times L$$

Donde:

  • \(d\): Tasa de demanda diaria.

  • \(L\): Lead Time del proveedor en días.

La Posición de Inventario: El error más común

El Arquitecto de Suministros nunca mira solo el inventario físico. Mira la Posición de Inventario:

$$\text{Posición} = \text{Físico} + \text{En Tránsito} - \text{Backorders}$$

Regla de Oro: El ROP debe compararse siempre contra la Posición de Inventario. Si solo miras el estante y olvidas lo que ya viene en el camión, duplicarás pedidos innecesariamente.

12.4 Simulador: El Modelo de Dientes de Sierra

Mientras que la gráfica anterior nos muestra el costo, esta simulación nos muestra el flujo físico del inventario en el tiempo. Observa cómo el inventario cae linealmente hasta tocar el ROP, disparando la orden que llega justo antes del quiebre.

sim_data = {
  const points = [];
  let current_stock = eoq;
  let arrival_day = -1;

  for (let t = 0; t <= 120; t++) {
    if (t === arrival_day) {
      current_stock += eoq;
      arrival_day = -1;
    }
    points.push({ day: t, stock: current_stock, rop: rop });
    current_stock -= d_daily;
    if (current_stock < 0) current_stock = 0;
    if (current_stock <= rop && arrival_day === -1) {
      arrival_day = t + lead_time;
    }
  }
  return points;
}

12.5 La Robustez del EOQ: ¿Qué pasa si me equivoco?

Una de las bellezas del EOQ es su insensibilidad al error. Debido a la raíz cuadrada, si sobreestimas el costo de pedido en un 100%, tu lote solo se desviará un 41%.

Esta “curva plana” (que puedes observar en la gráfica de Costo Total arriba) significa que es mejor tener un EOQ aproximado basado en lógica financiera que seguir usando reglas de dedo arbitrarias.
