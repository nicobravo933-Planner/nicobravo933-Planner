6. Suavizando la Curva: La Inteligencia Estadística

En el capítulo anterior, establecimos el “piso” de nuestra precisión con los modelos ingenuos. Aprendimos que si no podemos superar la simplicidad del modelo Naive, nuestra arquitectura está fallando. Pero, ¿qué pasa cuando la realidad exige algo más sofisticado que simplemente repetir el ayer, pero menos complejo que una red neuronal? Aquí es donde entra el Suavizamiento Exponencial.

6.1 El Concepto de la Memoria Selectiva

El Suavizamiento Exponencial Simple (SES) es el primer peldaño de la verdadera inteligencia en pronósticos. A diferencia del Promedio Móvil Simple (SMA), que otorga el mismo peso a todos los datos dentro de una ventana de tiempo (y olvida de golpe todo lo que queda fuera de ella), el SES utiliza una lógica de ponderación decreciente.

En este modelo, los datos más recientes son los más influyentes, y la importancia de los datos pasados se desvanece de forma geométrica a medida que retrocedemos en el tiempo. Matemáticamente, esto significa que el modelo nunca “olvida” del todo, pero su memoria se vuelve infinitesimalmente pequeña para eventos muy antiguos. Como Arquitectos de Suministros, nuestra misión es calibrar qué tan rápido debe el sistema “dejar ir” el pasado para capturar el presente.

6.2 La Anatomía del Parámetro $\alpha$ (Alpha)

La fórmula maestra del SES es una de las más elegantes de la estadística de suministros:

\[ \large \hat{y}_{t+1} = \alpha y_t + (1-\alpha) \hat{y}_t \]

Donde:

  • \(\alpha\) (Alpha) es el coeficiente de suavizamiento (un valor entre 0 y 1).

  • \(y_t\) es la demanda real observada en el periodo actual.

  • \(\hat{y}_t\) es el pronóstico que habíamos calculado para el periodo actual.

El Dilema del Arquitecto: Reactividad vs. Estabilidad

La elección del valor de \(\alpha\) es una decisión estratégica, no solo matemática:

  • Escenario de Alta Reactividad (\(\alpha \to 1\)): Si configuras un alfa de 0.9, el modelo se vuelve “nervioso”. Responderá casi instantáneamente a una promoción o a un pedido inesperado de un cliente grande. Esto es ideal para productos con ciclos de vida cortos o modas pasajeras, pero es peligroso en entornos con mucho ruido aleatorio, ya que el inventario “perseguirá” fantasmas estadísticos.

  • Escenario de Alta Estabilidad (\(\alpha \to 0\)): Con un alfa de 0.1, el modelo es “conservador”. Ignorará los picos repentinos de demanda, asumiendo que son anomalías. Esto protege la operación de cambios bruscos innecesarios, pero tiene un costo: si la demanda cambia de nivel permanentemente (un nuevo contrato, por ejemplo), el modelo tardará meses en darse cuenta, provocando quiebres de stock sistemáticos.

6.3 Simulador: El Laboratorio de Reactividad (SES)

Usa este simulador para ajustar el parámetro \(\alpha\) sobre una serie de tiempo ruidosa. Hemos incluido un “salto de nivel” a mitad de la serie para que puedas observar qué tan rápido (o lento) reacciona cada configuración. Observa cómo el MAE mide la precisión y el BIAS delata si te estás quedando corto tras el cambio de nivel.

jStat = require("jstat@1.9.6")
Plotly = require("plotly.js-dist-min@2.24.1")

viewof alpha = Inputs.range([0.01, 1], {value: 0.3, step: 0.05, label: "Parámetro Alpha (α):"})
viewof noise_level = Inputs.range([5, 50], {value: 20, step: 5, label: "Ruido de la Demanda:"})
ts_data = {
  const data = [];
  let current = 100;
  for (let i = 0; i < 70; i++) {
    const noise = jStat.normal.sample(0, noise_level);
    // Añadimos un salto de nivel estructural en el día 35
    const level_shift = i > 35 ? 40 : 0;
    data.push({ t: i, y: current + level_shift + noise });
  }
  return data;
}

// 2. Cálculo de SES
ses_calc = {
  const forecast = [];
  let last_f = ts_data[0].y; 
  
  for (let i = 0; i < ts_data.length; i++) {
    forecast.push(last_f);
    last_f = alpha * ts_data[i].y + (1 - alpha) * last_f;
  }
  return forecast;
}

// 3. Auditoría de Métricas con FVA
metrics = {
  const errors = ts_data.slice(1).map((d, i) => ses_calc[i+1] - d.y);
  const mae = jStat.mean(errors.map(e => Math.abs(e)));
  const bias = jStat.mean(errors);
  
  // Benchmark Naive (y_t = y_{t-1})
  const naive_errors = ts_data.slice(1).map((d, i) => Math.abs(d.y - ts_data[i].y));
  const mae_naive = jStat.mean(naive_errors);
  const fva = mae_naive - mae;

  return { mae, bias, fva };
}

// 4. Visualización
{

  const traces = [
    {
      x: ts_data.map(d => d.t),

      name: 'Demanda Real',

}
ImportantTablero de Control Estratégico

  • MAE (Precisión): unidades. Indica el error promedio por periodo.

  • BIAS (Dirección): (). Tras un salto de nivel, un BIAS muy negativo indica que el alfa es demasiado bajo.

  • FVA (Valor Agregado): pts sobre Naive. Si es negativo, el suavizamiento está “engañando” al sistema y sería mejor usar el dato del periodo anterior.

6.4 Más allá del SES: Evolución a Holt y Holt-Winters

El SES es el “caballo de batalla” para productos estables, pero tiene una debilidad fatal: es ciego ante la tendencia y la estacionalidad. Si tus ventas están subiendo un 5% cada mes, el SES siempre se quedará corto (BIAS negativo infinito). Para resolver esto, la arquitectura evoluciona añadiendo “capas” de inteligencia:

1. Modelo de Holt (Doble Suavizamiento Exponencial)

Añade un segundo parámetro, Beta (\(\beta\)), dedicado exclusivamente a estimar la pendiente (tendencia).

  • El modelo ahora tiene dos ecuaciones: una para el nivel y otra para la tendencia.

  • Si tienes un lanzamiento de producto exitoso, Holt “verá” la rampa de crecimiento y proyectará la línea hacia arriba, superando la limitación del SES.

2. Modelo de Holt-Winters (Triple Suavizamiento)

Es la corona de los modelos clásicos. Añade un tercer parámetro, Gamma ($\gamma$), para capturar la estacionalidad.

  • Es capaz de entender que vendes más los lunes que los miércoles, o más en diciembre que en enero.

  • Es el estándar de oro para productos de consumo masivo que se ven afectados por el calendario.

3. El Concepto de Amortiguación (Damped Trend)

Un error común de los modelos de tendencia es que proyectan un crecimiento infinito. En la vida real, nada crece para siempre. La Tendencia Amortiguada añade un factor que “frena” la pendiente a largo plazo, evitando que el sistema sugiera comprar inventario para un crecimiento que eventualmente se estancará.

6.5 Optimización: En busca del Alfa Perfecto

¿Cómo elegimos el mejor alfa para 10,000 SKUs? No lo hacemos a mano. Como arquitectos, diseñamos un proceso de optimización de parámetros:

  1. El software prueba cientos de combinaciones de alfas (y betas/gammas si aplica).

  2. Para cada combinación, mide el error histórico (generalmente minimizando el MSE o MAE).

  3. Selecciona la configuración que “mejor hubiera predicho el pasado”.

Este proceso se conoce como Ajuste de Parámetros y es la base de los motores de pronóstico modernos.
