15. El Laboratorio Virtual

Felicidades. Has completado la Parte IV, donde dominaste las leyes de la física del inventario. Aprendiste sobre el EOQ, el Stock de Seguridad y las distribuciones de probabilidad. Pero, como todo Arquitecto de Suministros sabe, hay un momento en que las fórmulas se quedan cortas.

¿Qué pasa si tu Lead Time no es constante ni sigue una curva normal? ¿Qué pasa si tienes restricciones de presupuesto que afectan a 500 productos a la vez? ¿Qué pasa si el costo de transporte cambia según el volumen del camión? Cuando las ecuaciones fallan, recurrimos a la herramienta definitiva: La Simulación de Monte Carlo.

15.1 De la Fórmula al Experimento

Hasta ahora, hemos usado fórmulas como \(SS = k \times \sigma \times \sqrt{L}\). Estas fórmulas son útiles, pero asumen un mundo “promedio”. La simulación, en cambio, no intenta predecir un resultado; intenta recrear el proceso.

En lugar de preguntar “¿Cuál es mi stock de seguridad?”, en este capítulo preguntamos: “Si opero con este stock durante los próximos 365 días bajo estas condiciones de caos, ¿cuántas veces me voy a quedar vacío?”.

15.2 ¿Qué es la Simulación de Monte Carlo?

El nombre proviene del famoso casino de Mónaco. La lógica es simple: si no puedes calcular la probabilidad de un evento complejo matemáticamente, genera miles de escenarios aleatorios y observa qué sucede con mayor frecuencia.

En Supply Chain, simulamos:

  1. Demanda Diaria: Generada aleatoriamente siguiendo el patrón que identificamos en la Parte II (Normal, Gamma o Poisson).

  2. Lead Time: El tiempo de llegada de cada pedido, que también varía aleatoriamente.

  3. Eventos Disruptivos: Cierres de puertos, huelgas o picos virales.

15.3 Por qué el Arquitecto prefiere simular

La simulación tiene tres ventajas críticas sobre las fórmulas:

  • Captura la Correlación: En la vida real, si el Lead Time es largo, la probabilidad de que la demanda suba durante ese tiempo es distinta. Las fórmulas ignoran esto; la simulación lo vive.

  • Maneja Restricciones: Puedes programar que un camión solo sale si está al 80% de capacidad. Intenta poner eso en una raíz cuadrada.

  • Visualiza el Riesgo de Cola: Te permite ver el “peor escenario posible”, no solo el promedio.

15.4 Simulador: Tu Primer Experimento de Monte Carlo

En este laboratorio, vamos a simular 100 días de operación. A diferencia de las gráficas de “dientes de sierra” perfectos del Capítulo 12, aquí cada día es una moneda al aire.

Ajusta la incertidumbre y observa cómo la línea de inventario se comporta de forma errática. ¿Cuántas veces toca el cero?

jStat = require("jstat@1.9.6")
Plotly = require("plotly.js-dist-min@2.24.1")

viewof avg_demand = Inputs.range([5, 50], {value: 20, step: 1, label: "Demanda Promedio Diaria:"})
viewof sigma_demand = Inputs.range([0, 30], {value: 10, step: 1, label: "Variabilidad Demanda (σ):"})
viewof lt_mean = Inputs.range([1, 15], {value: 5, step: 1, label: "Lead Time Promedio (días):"})
viewof lt_sigma = Inputs.range([0, 5], {value: 2, step: 0.5, label: "Variabilidad Lead Time (σ):"})
viewof initial_stock = Inputs.range([50, 500], {value: 150, step: 10, label: "Stock Inicial:"})
viewof reorder_point = Inputs.range([50, 400], {value: 120, step: 10, label: "Punto de Reorden (ROP):"})
viewof order_qty = Inputs.range([100, 1000], {value: 300, step: 50, label: "Cantidad de Pedido (Q):"})
sim_results = {
  const days = 100;
  let stock = initial_stock;
  let pending_orders = []; // [{arrival_day, qty}]
  const history = [];

  for (let d = 0; d < days; d++) {
    // 1. Llegada de pedidos pendientes
    let arrivals = 0;
    pending_orders = pending_orders.filter(o => {
      if (o.arrival_day <= d) {
        arrivals += o.qty;
        return false;
      }
      return true;
    });
    stock += arrivals;

    // 2. Generar demanda aleatoria (Truncada a 0)
    const demand = Math.max(0, jStat.normal.sample(avg_demand, sigma_demand));
    const sold = Math.min(stock, demand);
    const stockout = demand > stock ? demand - stock : 0;
    
    stock -= sold;

    // 3. Revisión de política (ROP)
    if (stock <= reorder_point && pending_orders.length === 0) {
      const actual_lt = Math.max(1, Math.round(jStat.normal.sample(lt_mean, lt_sigma)));
      pending_orders.push({
        arrival_day: d + actual_lt,
        qty: order_qty
      });
    }

    history.push({ day: d, stock: stock, stockout: stockout, demand: demand });
  }
  return history;
}

15.5 Interpretando el Caos: El Intervalo de Confianza

Cuando corres una simulación de Monte Carlo, no obtienes un número, obtienes una distribución de resultados. Si repitieras esta simulación 1,000 veces:

  1. Verías que algunas veces el stock nunca toca el cero.

  2. Otras veces, por una mala racha de demanda alta y proveedor lento, quedarías vacío por semanas.

Tu trabajo como Arquitecto no es evitar el quiebre a toda costa, sino dimensionar el riesgo. Si la simulación te dice que el “peor escenario” (el percentil 95) es perder $10,000 en ventas, y prevenirlo te cuesta $15,000 en inventario, ya sabes qué decisión tomar.
